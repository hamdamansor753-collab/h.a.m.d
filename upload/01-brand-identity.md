# 01 - الهوية التجارية (Brand Identity)

> ملاحظة: هذا الملف مرجع تصميمي لأي كود UI يُولَّد. أي مخرجات من GLM لواجهة المستخدم يجب أن تلتزم بالقواعد هنا حرفيًا.

## اسم البراند
H.A.M.D

## نظام الألوان (Design Tokens)
> [يُستكمل بالقيم الفعلية من نظام H.A.M.D الحالي — navy/cyan dark theme]

| المتغير | الاستخدام |
|---|---|
| `--color-primary` (navy) | الخلفيات الأساسية، الهيدر |
| `--color-accent` (cyan) | الأزرار الأساسية، الروابط، العناصر التفاعلية |
| `--color-surface` | كروت وبطاقات المحتوى |
| `--color-text-primary` / `--color-text-secondary` | النصوص |
| `--color-success` / `--color-warning` / `--color-danger` | حالات النظام (مدفوع/معلق/مرفوض) |

## الخطوط
- **العربي**: Cairo (كل الأوزان: Regular, Medium, SemiBold, Bold)
- **الإنجليزي**: خط مكمّل يتماشى بصريًا مع Cairo (يُحدَّد لاحقًا لو لازم)

## اتجاه الواجهة (RTL/LTR)
- الافتراضي: RTL للعربي
- كل مكوّن UI **لازم** يُبنى بمنطق CSS logical properties (`margin-inline-start` لا `margin-left`) لضمان التبديل السليم بين RTL/LTR بدون كسر التصميم
- لا يُسمح بأي قيم اتجاه ثابتة (hardcoded left/right) في أي كومبوننت

## نغمة الكتابة (Tone of Voice)
- عربي فصيح مبسّط في الواجهة الرسمية (فواتير، تقارير)
- عربي مصري/خليجي مقبول في الرسائل التفاعلية (تنبيهات، أخطاء ودّية) — يُحدَّد حسب لغة المستخدم المختارة
- لا نصوص إنجليزية مباشرة داخل الواجهة العربية إلا للمصطلحات التقنية التي لا ترجمة عربية شائعة لها

## معيار الالتزام بالهوية (لِـ GLM)
أي كومبوننت UI جديد يُنتَج يجب أن:
1. يستخدم design tokens فقط (لا قيم ألوان hardcoded)
2. يدعم RTL/LTR تلقائيًا
3. يستخدم Cairo font كخط افتراضي
4. يمرّر كل النصوص الظاهرة للمستخدم عبر نظام الترجمة (i18n keys)، بدون استثناء
